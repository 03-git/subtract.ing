# lineage.txt

The people whose code subtract depends on.

---

## Brian Fox — the substrate author

Announced bash as a beta in June 1989 (Fox 1989a). Primary maintainer
until early 1993, when Chet Ramey took over. Authored the readline
library (Fox 1989b), the history library (Fox 1989c), and GNU Info.

Three things Fox wrote that subtract depends on directly:

**bash itself.** The dispatcher. `command_not_found_handle`. `case`.
Parameter expansion. Process substitution. Every tier of subtract's
routing is a bash construct Fox authored. subtract is GPL 3.0 because
bash is GPL 3.0.

**readline** (Fox 1989b). Tab completion, `Ctrl-R` history search,
line editing. When T0 consults `compgen` for a completion candidate,
the reason `compgen` has anything to say is that Fox made completion
a first-class shell primitive in 1989.

**the history library** (Fox 1989c). `!!`, `!$`, `Ctrl-R`, `HISTFILE`.
Episodic memory for the shell, shipped in 1989, present in every bash
install since.

## Chet Ramey — the substrate steward

Took over bash maintenance from Fox in 1993 and has held the seat
for thirty-three years. Authored `help` as a builtin in the form
subtract's T0.5 tier consults (`help -m`, `help -s`). Kept
`command_not_found_handle` a stable public hook rather than letting
it become an implementation detail.

Fox wrote the substrate. Ramey kept it reachable. subtract needs both.
