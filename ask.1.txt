.TH ASK 1 "2026-04-13" "subtract.ing" "User Commands"
.SH NAME
ask \- query your local model (Claude without the browser)
.SH SYNOPSIS
.B ask llama.cpp
.I query
.br
.B ask curl
.I query
.SH DESCRIPTION
.B ask
sends a question to an inference backend and prints the answer.
You name the backend in the command.
.PP
If you are here, you probably got tired of browser latency, rate limits,
or paying per token. This is the same capability, minus the browser.
.SH BACKENDS
.TP
.B ask llama.cpp
Local inference via llama.cpp server. No network. No account. No rate limit.
Requires llama.cpp running locally (see
.BR llama.cpp (1)).
.TP
.B ask curl
External API call. Crosses a network boundary you do not control.
Use for live data or when you intentionally want a remote model.
.TP
.B ask local
Alias for
.BR "ask llama.cpp" .
.SH EXAMPLES
Query local model:
.PP
.RS
.nf
ask llama.cpp "what system call creates a child process?"
.fi
.RE
.PP
Query external API:
.PP
.RS
.nf
ask curl "current weather in Chicago"
.fi
.RE
.PP
Use the alias:
.PP
.RS
.nf
ask local "how do I resize an image?"
.fi
.RE
.SH DISCOVERY
Many queries do not need inference. subtract routes definitional
queries to
.BR apropos (1)
first:
.PP
.RS
.nf
"what compresses files?"
  \(-> apropos: gzip, bzip2, xz
.fi
.RE
.PP
Faster, offline, free. If apropos answers, the model is not invoked.
.SH WHY THE BACKEND IS IN THE COMMAND
The command names its mechanism. This matters when:
.IP \(bu 2
Debugging: the command shows where it routes
.IP \(bu 2
Teaching: a child sees exactly what is invoked
.IP \(bu 2
Trust: a command that names its primitive cannot lie
.PP
.B ask chatgpt
would hide whether it hits a local model or a remote API.
.B ask llama.cpp
cannot lie about what it does.
.SH PLANNED
A future version will support bare
.B ask
.I query
with auto-detection of the local backend (ollama, llama.cpp, lm-studio).
Until then, name the backend explicitly.
